# Руководство по участию в разработке

## Как сообщать об ошибках

1. Проверьте, не было ли уже такого Issue
2. Используйте шаблон:
   - **Версия игры**: 
   - **ОС**: 
   - **Описание проблемы**: 
   - **Шаги воспроизведения**: 
   - **Ожидаемое поведение**: 
   - **Скриншоты/видео**: 

## Как предлагать улучшения

1. Создайте Issue с меткой `enhancement`
2. Опишите предлагаемую фичу
3. Объясните, почему это улучшит игру
4. Приложите примеры реализации (если есть)

## Pull Request процесс

1. Форкните репозиторий
2. Создайте ветку `feature/название-фичи`
3. Внесите изменения
4. Проверьте, что игра запускается без ошибок
5. Создайте Pull Request в ветку `main`
6. Дождитесь ревью

## Стиль кода

- Используйте 4 пробела для отступов
- Имена переменных и функций в snake_case
- Имена классов в PascalCase
- Комментируйте сложные участки кода
